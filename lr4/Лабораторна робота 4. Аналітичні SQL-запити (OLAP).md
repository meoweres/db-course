### Цілі:
---
 - **4 запити**, що містять агрегаційні функції (`SUM`, `AVG`, `COUNT`, `MIN`, `MAX`, `GROUP BY`);
 - **3 запити**, що використовують різні типи джоінів (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `FULL JOIN`, `CROSS JOIN`);
 - **3 запити** з використанням підзапитів (вибірка з підзапитом у `SELECT`, `WHERE`, або `HAVING).

### Примітка:
---
Оскільки за умовами даної лабораторної роботи звіт пишеться у форматі markdown, то замість скріншотів результатів запитів будуть приведені запити з коментарями та виведення цих запитів.


### Запити:
---
#### З агрегаційними функціями:
---
1) Запит кількості наявних книг (**BookNamesAmount.sql**)
```
select count(*) as "Кількість книг" from Books;
```
Тут підраховуються всі рядки в таблиці `Books`, а при виводі стовпець отримує псевдонім "Кількість книг".

2) Кількість всіх приміркників всіх книг (**BooksAmount.sql**)
```
select sum(quantity) as "Кількість всіх примірників" from books;
```
Тут рахуються всі комірки колонки кількості (`quantity)` в таблиці `Books`, та надається псевдонім при виводі.

3) Мінімальний та максимальний рік книг (**MinMaxBookYear.sql**)
```
select min(year) as "Мінімальна кількість примірників", max(year) as "Максимальна кількість примірників" from books;
```
Тут виводяться дані про найновіші та найстаріші книги в асортименті таблиці `Books`.


#### Різнотипові джоіни:
---
1) Звичайний `INNER JOIN`. Книги з жанрами (**BooksWithGenres.sql**)
```
select b.id as "ІД Книги",
b.title as "Назва книги",
string_agg(g.genre::text, ', ') as "Жанри"
from Books b
join Genres g on b.id = g.book_id
group by b.id, b.title
order by b.id;
```
Цей запит видає тільки книги, яким є назначений жанр (запис в таблиці `Genres`), оскільки `INNER JOIN` працює тільки при наявності записів в обох таблицях водночас. Виводиться ідентифікатор книги, її назва, та жанри. Жанри об'єднуються в одну комірку та розділені комами.

2) `LEFT JOIN`. Перелік читачів за позиками та без (**ReadersWithAndWithoutLoans.sql**)
```
select r.id as "ІД",
r.name as "Ім'я",
r.surname as "Призвіще",
string_agg(distinct l.id::text, ', ') as "ІД позики",
string_agg(distinct b.title::text, ', ') as "Всі позичені книги"
from Readers r
left join Loans l on r.id = l.reader_id
left join BookLoans bl on l.id = bl.loan_id
left join Books b on bl.book_id = b.id
group by
r.id,
r.name,
r.surname
order by r.id asc;
```
Цей запит видає список читачів з таблиці Readers та всі книги, що вони хоча б раз позичали. Виводиться `id` позики та назви книг, обидві ці позиції перелічуються з роздільником комою. Дублікати не включаються, завдяки `DISTINCT`. За відсутності даних для читача в комірці буде `[null]`. Для доступу до назви, запит йде від `id` читача (Readers) до його `id` позики (`Loans`), звідти йде до `BookLoans` та бере`id` книги, з чим звертається до таблиці `Books` за його назвою (`Books(title)`).

3) `FULL JOIN`. Перелік всіх книг з авторами та видавництвами (**AllBooksAuthorsPublishers.sql**)
```
select b.title as "Назва книги",
a.fullname as "Автор",
p.name as "Видавництво"
from Books b
full join BookAuthorPublishers bap on b.id = bap.book_id
full join Authors a on bap.author_id = a.id
full join Publishers p on bap.publisher_id = p.id
order by a.id;
```
Такий запит виводить всі існуючі позиції з таблиць книг (`Books`), авторів (`Authors`) та видавництв (`Publishers`). Якщо дані для відповідних позицій відсутні, то там буде `[null]`. Запит перевіряє зв'язки через проміжну таблицю зв'язків між книгою, автором та видавництвом (`BookAuthorPublishers`).
#### З підзапитами:
---
1) Запит на читачів, які надали електронну пошту **ReadersWithEmail.sql**
```
select names as "Ім'я",
surname as "Прізвище",
email as "Ел.пошта"
from Readers
where email is not null 
order by id;
```
Цей запит видає ім'я, прізвище та адресу електронної пошти тільки тих читачів, що надали її при реєстрації.

2) Запит на книги з великою кількістю наявних примірників (**BooksBigQuantity.sql**)
```
select title as "Назва книги", quantity as "Кількість примірників"
from Books
where quantity >= 10
order by quantity desc;
```
Цей запит виведе дві колонки: в одній назва книги, а в іншій - кількість примірників. При чому, всі книги в наявності у кількості 10 штук або більше. Вивід сортується по кількості примірників - від найбільшої до найменшої.

3) Запит на читачів, що робили більше за одну позику (**RegularReaders.sql**)
```
select
r.id as "ІД Читача",
r.name as "Ім'я читача",
r.surname as "Прізвище читача",
count(l.id) as "Кількість позик"
from Readers r
join Loans l on r.id = l.reader_id
group by
r.id,
r.name,
r.surname
having count(l.id) > 1;
```
Цей запит видає читачів, що брали книги більше 1 раза в бібліотеці. Виводиться ід, ім'я, прізвище та кількість разів.

---
