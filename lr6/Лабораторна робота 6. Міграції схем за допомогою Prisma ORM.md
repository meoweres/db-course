## Цілі
---
- Використати Prisma ORM для керування схемами та дослідити, як Prisma може аналізувати та змінювати схему вашої бази даних.
- Застосування міграцій - генерування та застосування змін схеми (таблиць, стовпців, зв'язків) за допомогою `prisma migrate`.
- Моделювання за допомогою файлів схеми Prisma - визначення таблиць та зв'язків у `schema.prisma` та перегляд їхнього відображення в PostgreSQL.
- Виконати базові запити Prisma - вставити та запитати дані за допомогою клієнта Prisma (через _Prisma Studio_ або простий скрипт) для перевірки змін.

### Внесення змін до схеми

Примітка:
Перша міграція була створена для фіксування стану таблиці до змін через Prisma ORM.
#### Нова таблиця
Створена нова таблиця Ratings та доданий зв'язок до таблиці книг Books. Мета нової таблиці - зберігати оцінку книги. 

До:
```
model books {
  id                   Int                    @id @default(autoincrement())
  title                String
  year                 Int                    @db.SmallInt
  quantity             Int
  bookauthorpublishers bookauthorpublishers[]
  bookloans            bookloans[]
  genres               genres[]
}
```

Після:
```
model ratings {
  book_id Int
  rating Int
  books   books  @relation(fields: [book_id], references: [id], onDelete: Cascade, onUpdate: NoAction)

  @@id([book_id, rating])
}

model books {
  id                   Int                    @id @default(autoincrement())
  title                String
  year                 Int                    @db.SmallInt
  quantity             Int
  bookauthorpublishers bookauthorpublishers[]
  bookloans            bookloans[]
  genres               genres[]
  ratings              ratings[]
}
```

#### Зміна існуючої таблиці
Зміна була зроблена в таблиці книг Books. В якості зміни була додана нова колонка для опису книги (description), обмежений в 500 символів.

#### Перевірка даних
У скріншоти.pdf наведені скріншоти з Prisma Studio, зо показують, що дії над таблицею були проведені та підтверджена працездатність після змін, перелічених вище. Для зручності продублюю запит, яким перевірялися таблиці на скріншоті 3. Самі дані були вставлені руками через інтерфейс.

```
select b.title as "Назва книги",
  a.fullname as "Автор",
  r.rating as "Рейтинг",
  b.description as "Опис книги"

from Books b
left join Ratings r on b.id = r.book_id
left join bookauthorpublishers bap on b.id = bap.book_id
left join Authors a on bap.book_id = a.id
order by b.id asc;
```
